二、什么是 AOP

2.1 AOP 概述

AOP ( Aspect Orient Programming ) 面向切面编程。

2.2 AOP 相关术语

  • Aspect：切面，
  • JoinPoint：连接点，
  • Pointcut：切入点，
  • 目标对象：给哪个类的方法添加功能，这个类就是目标对象。
  • Advice：通知，表示切面功能执行的时机。

2.3 AOP 实现框架

aop 是一个规范，是动态代理的一个规范化、一个标准。

  • spring：spring 在内部实现了 aop 规范，能做 aop 的工作（少用，笨重）
    • 库：org.spring.framework:spring-aop:5.x.x.RELEASE.jar
  • aspectJ: 一个开源的专门做 aop 的框架（spring 已集成）
    • 官网：https://www.eclipse.org/aspectj/

三、aspect 的5个通知注解

3.1 切面执行时机

  • @Before
  • @AfterReturning
  • @Around
  • @AfterThrowing
  • @After

3.2 切入点表达式

  • execution(public * *(…))
    • 指定切入点为：任意公共方法。
  • execution(* set*(…))
    • 指定切入点为：任意一个以 set 开始的方法。
  • execution(* com.g3g4x5x6.service.*.*(…))
    • 指定切入点为：com.g3g4x5x6.service 包中的任意类的任意方法
  • execution(* com.g3g4x5x6.service…*.*(…))
    • 指定切入点为：service 包或者子包中的任意类的任意方法
  • execution(* *…service.*.*(…))
    • 指定切入点为：所有包下的 service 子包下的所有类（接口）中的任意方法
