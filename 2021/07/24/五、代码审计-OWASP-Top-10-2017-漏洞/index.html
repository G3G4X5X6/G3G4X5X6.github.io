五、代码审计 “OWASP Top 10 2017” 漏洞

本文最后更新于：2021年7月28日 上午

OWASP( Open Web Application Security Project，开放式 Web 应用程序安全项目 ) 是一个组织，它提供有关计算机和和互联网应用程序的公正、实际、有成本效益的信息，其目的是协助个人、企业和机构来发现和使用可信赖软件。其中 OWASP Top 10 （十大安全漏洞列表）颇具权威性。

0x00 注入

0x0000 注入漏洞简介

将不受信任的数据作为命令或查询的一部分发送到解析器时，会产生诸如 SQL 注入、NoSQL 注入、OS 注入和LDAP 注入的注入缺陷。攻击者的恶意数据可以诱使解析器在没有适当授权的情况下执行非预期命令或访问数据。

0x0001 SQL 注入

SQL注入攻击通过构建特殊的输入作为参数传入Web应用程序，而这些输入大都是未经过滤直接拼接在 SQL 语句中，通过执行SQL语句进而执行攻击者所要的操作，它目前是黑客对数据库进行攻击的最常用手段之一。

1. JDBC 拼接不当造成SQL注入

  • 使用了 PrepareStatement 也不一定安全，因为可能在预编译前就拼接了 SQL 语句。

2. 框架使用不当造成 SQL 注入

  • MyBatis 框架
    • # 与 $ 的区别
      • 使用 #{Parameter} 方式会进行预编译。
      • 使用 ${Parameter} 方式会进行字符串拼接，存在 SQL 注入的风险。
  • Hibernate 框架（Java 持久化 API，JPA）
    • 采用 Hibernate 查询语言（HQL）注入
    • HQL 与 SQL 类似，但有些许不同。受语法的印象，HQL 注入在实际漏洞利用上具有一定的限制。
    • 正确使用以下几种 HQL 参数绑定的方式可以有效避免注入的产生。
      • 位置参数 (Positional parameter)
      • 命名参数 (Named parameter)
      • 命名参数列表 (Named parameter list)
      • 类实例 (JavaBean)
  • 防御不当造成 SQL 注入
    • SQL 注入最重要的成因在于未对用户输入进行严格有效的过滤，并采取不恰当的方式构造 SQL 语句。

0x0002 命令注入

命令注入（Command Injection）是指在某种开发需求中，需要引入对系统本地命令的支持来完成某些特定的功能。。当未对可控输入的参数进行严格有效的过滤时，则有可能发生命令注入。

Runtime.getRuntime().exec(‘whoami’)

1. 命令注入的局限性

  • 恶意输入：www.baidu.com&ipconfig，拼接出系统命令为：ping www.baidu.com&ipconfig
  • 在操作系统中该命令成功执行，然而在 Java 运行环境下，却执行失败。
  • 原因是在Java程序的处理中，ping www.baidu.com&ipconfig 被当作一个完整的字符串而非两条命令。

0x0003 代码注入

产生代码注入漏洞的前提条件是将用户输入的数据作为 Java 代码进行执行。

代码注入（Code Injection）与命令注入相似，指在正常的 Java 程序中注入一段 Java 代码并执行。相比于命令注入，代码注入更具灵活性，注入的代码可以写入或修改系统文件，甚至可以直接注入执行系统命令的代码。

  • Apache Commons Collections 组件 3.1 版本利用反射来完成特定功能的代码。

2. EL 基础语法

  • 在 JSP 中， 用户可以使用 ${} 来表示此处为 EL 表达式，例如，表达式 ${name} 表示获取 name 变量。
  • 当 EL 表达式未指定作用域范围时，默认在 page 作用域范围内查找，而后依次在request、session、application 范围查找。
  • 也可以指定作用域查找，如：${requestScope.name}

3. 获取对象属性

EL 表达式有两种方式获取对象属性的方式：

  • 第一种：${对象.属性}
  • 第二种：${param[name]}
